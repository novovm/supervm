# SuperVM 双轨定价机制 - 刚性兑付 + 市场交易

> **灵感来源**: 创始人提出的"双重价格体系"概念  
> **日期**: 2025-11-17  
> **核心思想**: 用户可选择"按净资产刚性兑付"或"市场价自由交易"

---

## 🎯 核心概念

您的构想完美结合了**央行式刚性兑付**和**市场化自由交易**,创造了双轨价格体系:

```
SuperVM Token 的两个价格:

价格 A: 净资产价格 (NAV - Net Asset Value)
= 外汇储备总价值 / Token 流通量
= 刚性兑付价格 (类似基金净值)

价格 B: 市场价格 (Market Price)
= AMM 池供需关系
= 自由浮动价格

用户选择:
├─ 刚性兑付: 按净资产价格向国库赎回 → 稳定但慢
└─ 市场交易: 按市场价格在 AMM 卖出 → 快速但波动
```

## 💎 双轨定价详解

### 1. 净资产价格 (NAV) - 刚性兑付

#### 计算公式

```
净资产价格 (NAV) = 总储备价值 / 流通量

示例计算:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
外汇储备:
- BTC: 100 × $50,000 = $500 万
- ETH: 2,000 × $3,000 = $600 万
- USDT: 200 万 = $200 万
- BNB: 10,000 × $300 = $300 万
- 总储备价值: $1,600 万

流通量: 50,000,000 Token

NAV = $16,000,000 / 50,000,000 = $0.32/Token
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 刚性兑付机制

```rust
// 用户向国库申请赎回
pub fn redeem_at_nav(
    user: Address,
    token_amount: u128,
) -> Result<RedemptionReceipt> {
    // 1. 计算当前 NAV
    let total_reserve_usd = calculate_total_reserve_value();
    let circulating_supply = get_circulating_supply();
    let nav = total_reserve_usd / circulating_supply;
    
    // 2. 计算应得外币
    let usd_value = token_amount * nav;
    
    // 3. 用户选择兑换的币种(默认 USDT)
    let currency = user.preferred_currency; // USDT/BTC/ETH
    
    // 4. 从储备池转出
    let amount = usd_value / get_price(currency);
    transfer_from_reserve(currency, user, amount)?;
    
    // 5. 销毁用户的 Token
    burn(user, token_amount)?;
    
    // 6. 返回凭证
    Ok(RedemptionReceipt {
        user,
        token_burned: token_amount,
        currency,
        amount_received: amount,
        nav_price: nav,
        timestamp: now(),
    })
}
```

#### 刚性兑付的特点

```
优势:
✓ 价格稳定(基于真实储备)
✓ 保本(永远不低于 NAV)
✓ 透明可预测
✓ 类似基金赎回

劣势:
✗ 需要等待期(如 T+7 赎回)
✗ 可能有赎回限额(防止挤兑)
✗ 错过市场溢价(如果市场价 > NAV)
```

### 2. 市场价格 (Market Price) - AMM 交易

#### AMM 池定价

```
Token/USDT AMM 池:
x (Token) × y (USDT) = k (恒定乘积)

示例:
初始: 10M Token × 5M USDT = 50M k
初始市场价: 5M / 10M = $0.5/Token

对比:
- NAV 价格: $0.32 (储备支撑)
- 市场价格: $0.5 (供需决定)
- 溢价: 56% (市场看好生态)
```

#### 市场交易特点

```
优势:
✓ 即时成交(秒级)
✓ 无限额限制
✓ 可能获得溢价收益
✓ 24/7 交易

劣势:
✗ 价格波动(可能暴涨暴跌)
✗ 滑点(大额交易)
✗ 可能跌破 NAV(恐慌抛售)
```

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
债券名称: SVM Bond Year 1

到期收益: 100 + 8 = 108 USDT

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 债券用途

```
1. 国库融资
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
发行 1000 万美元债券:
用途:
├─ 生态投资: 500 万(投资 DApp)
├─ 基础设施: 300 万(节点补贴)
└─ 流动性管理: 200 万(AMM 池注入)

优势:
✓ 不稀释 Token(相比直接增发)
✓ 固定成本(8% 利息 vs 不确定的增发压力)
✓ 提供稳定收益产品(吸引保守投资者)

2. 货币政策工具
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
通胀时: 发行债券 → 回收 Token → 减少流通
通缩时: 回购债券 → 释放 Token → 增加流通

类似美联储的公开市场操作!

3. 信用扩张基础
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
债券作为抵押品:
用户抵押 SVM Bond → 借出 Token
抵押率: 120% (债券面值的 83%)

示例:
持有 100 万 USDT 债券
→ 可借出 83 万 Token
→ 用于流动性挖矿/质押等
```

## 🏗️ 抵押铸币 (CDP) - 其它币种

### 多币种抵押铸币

您提到的"其它币种抵押主链债铸币"类似 MakerDAO 的 CDP 模型:

```
抵押资产层级:

Level 1: 主链 Token 本身
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
抵押: SVM Token
铸造: SVM-USD 稳定币
抵押率: 150%
示例: 抵押 150 SVM → 铸造 100 SVM-USD

Level 2: 主链债券
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
抵押: SVM Bond
铸造: SVM Token
抵押率: 120% (债券更安全)
示例: 抵押 120 USD 债券 → 铸造 100 SVM

Level 3: 外部资产(RWA)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
抵押: BTC/ETH/房产证/股票
铸造: SVM-USD 稳定币
抵押率: 200% (外部资产风险高)
示例: 抵押 200 USD BTC → 铸造 100 SVM-USD
```

### CDP 合约实现

```rust
pub struct CollateralizedDebtPosition {
    /// CDP ID
    pub cdp_id: u64,
    /// 抵押资产类型
    pub collateral_type: CollateralType,
    /// 抵押数量
    pub collateral_amount: u128,
    /// 债务数量(铸造的稳定币)
    pub debt_amount: u128,
    /// 所有者
    pub owner: Address,
    /// 创建时间
    pub created_at: u64,
}

pub enum CollateralType {
    SVMToken,      // 主链 Token
    SVMBond,       // 主链债券
    BTC,           // 比特币
    ETH,           // 以太坊
    RWA(String),   // 现实世界资产(房产/股票/黄金)
}

pub trait CDPManager {
    /// 创建 CDP(抵押铸币)
    fn create_cdp(
        &mut self,
        owner: Address,
        collateral_type: CollateralType,
        collateral_amount: u128,
        debt_amount: u128,
    ) -> Result<u64>; // 返回 cdp_id
    
    /// 增加抵押
    fn add_collateral(
        &mut self,
        cdp_id: u64,
        amount: u128,
    ) -> Result<()>;
    
    /// 铸造更多稳定币
    fn mint_more(
        &mut self,
        cdp_id: u64,
        amount: u128,
    ) -> Result<()>;
    
    /// 还款销毁
    fn repay_debt(
        &mut self,
        cdp_id: u64,
        amount: u128,
    ) -> Result<()>;
    
    /// 清算(抵押率不足)
    fn liquidate(
        &mut self,
        cdp_id: u64,
    ) -> Result<LiquidationResult>;
}
```

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
