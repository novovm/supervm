# SuperVM 主链经济白皮书 (草案)

> 版本: v0.1  
> 日期: 2025-11-17  
> 状态: 对外草案 / 投资人与合作方预览

---

## 1. 项目概述

### 1.1 愿景

SuperVM 的目标,是在一个高性能 WASM 优先( WASM-first )的区块链虚拟机之上,构建下一代**可编程金融基础设施**。

在这个体系中,主链 Token 不再只是“Gas 燃料费”,而是与一整套金融模块共同组成的 **去中心化金融系统**,综合扮演:

- **去中心化央行**: 负责发行、储备管理与货币政策
- **去中心化商业银行**: 提供存款、借贷与稳定币服务
- **去中心化投资银行**: 发行主链债券、资产证券化、RWA 对接
- **去中心化基金**: 通过 NAV 净值管理、分红、回购与销毁实现长期价值增长

### 1.2 设计原则

- **健全货币(Sound Money)**: 固定 M0 上限、长期通缩压力、公开透明的释放曲线
- **储备支撑(Reserve-Backed)**: BTC/ETH/USDT/BNB/SOL 多币种储备 + 现实世界资产(RWA) 做背书
- **市场定价(Market-Based)**: 通过 AMM 池完成价格发现,不采用刚性汇率锚定,价格由市场决定
- **现金流导向(Cashflow Oriented)**: 协议收入按规则每天分红给持币者,形成“股息型资产”
- **治理就绪(Governance Ready)**: 9 人加权议会 + 小户委托投票,兼顾效率与去中心化
- **跨链互操作(Interoperable)**: 支持多链支付、跨链结算、与 RWA 协议桥接

## 2. 货币层级与供给

### 2.1 三层货币结构

SuperVM 采用三层货币结构:

- **M0 – 基础货币(Base Money)**  
  固定上限的主链 Token (总量 100,000,000),一次性铸造,永不增发

- **M1 – 流通货币(Circulating Money)**  
  已解锁的 M0 减去已销毁部分,是市场上真实可流通的 Token 数量

- **M2 – 信用货币(Credit Money)**  
  在 M1 基础上,叠加所有稳定币与信用工具(如 CDP 债务头寸)

形式化表达:

- M1 = 已解锁的 M0 + 重新分配的手续费 − 已销毁 Token
- M2 = M1 + 稳定币(例如基于 SuperVM 的 USD 稳定币) + 各类 CDP 债务

### 2.2 初始分配与解锁

- M0 总量: 100,000,000 枚,一次性铸造,协议层面永不增发
- 典型分配方案(示例):
  - 40% 挖矿/节点激励(10 年线性释放)
  - 20% 生态基金(由治理控制支出)
  - 15% 团队(2 年锁定 + 3 年线性解锁)
  - 10% 早期投资人(1 年锁定 + 2 年线性解锁)
  - 10% 国库储备(用于外币支付替代发放给矿工的 Token)
  - 5% 流动性引导(用于初始 AMM 流动性)

具体分配比例可在治理阶段调整,以上为对外说明的典型结构。

### 2.3 通缩机制

SuperVM 通过多种方式实现长期通缩压力:

- **Gas BaseFee 销毁**: 每笔交易的基础 Gas 费用中,固定比例自动销毁
- **手续费销毁**: 每笔交易/服务手续费的一部分被销毁
- **清算罚金销毁**: CDP 清算时,部分罚金被直接销毁
- **国库回购销毁**: 在特定市况下,由治理决议,国库用收入回购 Token 并销毁

目标区间: 长期看,**净通缩率控制在每年 -3% ~ -5%** 左右,并根据流通量与经济活跃度动态调整。

## 3. 双轨定价与净资产刚性兑付

### 3.1 一枚 Token 的两个价格

SuperVM 引入“**双轨定价模型**”,即:

1. **市场价格 P_market**  
   - 由 AMM 池(如 Token/USDT、Token/BTC、Token/ETH 等)决定
   - 完全市场化、24/7 自由交易,由供需决定

2. **净资产价格 P_NAV** (Net Asset Value)  
   - 定义为:

$$
P_{NAV} = \frac{\text{储备资产总价值(折算 USD)}}{\text{流通供应量 M1}}
$$

   - 储备资产包括: BTC、ETH、USDT、BNB、SOL 以及未来引入的 RWA 资产组合

有了 P_NAV,系统为 Token 建立了一个**“软底价”**——在存在刚性兑付机制的前提下,理性参与者不会愿意在远低于 P_NAV 的价格大量抛售。

### 3.2 净资产刚性兑付机制

持币者可以选择按照 NAV 价格,向国库申请赎回(刚性兑付),前提条件包括:

- 每日赎回额度上限(如:每日最多赎回当前流通量的 1%),防止“挤兑”式攻击
- 结算周期(如:T+7 日到账),给系统调整与流动性调度时间
- 储备率阈值(如:当储备率 < 50% 时,暂停 NAV 赎回功能)

赎回流程示意:

1. 用户提交赎回申请(数量 + 期望外币种类,如 USDT/BTC/ETH)
2. 协议根据链上储备与预言机价格,计算当前 P_NAV
3. 按 P_NAV 计算应兑付的外币等值
4. 国库在 T+7 内向用户支付对应外币,并销毁用户赎回的 Token

这样,每一次 NAV 赎回都在**提高剩余 Token 的单位资产背书**。

### 3.3 套利与价格收敛

当市场价与 NAV 出现明显偏离时,套利者的行为会促使二者自然收敛:

- 如果 P_market ≪ P_NAV:
  - 套利者在 AMM 以低价买入 Token
  - 再按 P_NAV 向国库申请赎回
  - 获得无风险价差 P_NAV − P_market
  - 买入行为会推高 P_market,直至接近 P_NAV

- 如果 P_market ≫ P_NAV:
  - 无法通过 NAV 形成无风险套利
  - 但自然的获利了结行为 + CDP 提供的套现/加杠杆通道,会对价格形成向下压力

最终,**P_market 与 P_NAV 会在一个动态区间内来回震荡,但长期保持相对收敛**,实现“市场波动 + 资产背书”的平衡。

## 4. 主链债券(SVM Bonds)

### 4.1 债券在体系中的角色

SuperVM 引入原生链上债券(简称 **SVM Bonds**),承担三大职能:

1. **国库融资工具**: 为生态发展与基础设施提供长期、低摊薄的资金来源
2. **货币政策工具**: 通过发行/回购债券,实现流动性收缩与扩张
3. **高质量抵押品**: 作为 CDP 与借贷系统中的优质抵押资产

### 4.2 债券参数示例

- 面值: 100 USDT
- 发行价: 95 USDT (5% 折价)
- 期限: 1 / 3 / 5 年
- 票面利率: 8% / 12% / 15% 年化
- 利息支付: 每季度以主链 Token 支付
- 本金兑付: 到期后以 USDT(或其它稳定储备资产)一次性兑付

### 4.3 募集资金用途

- 生态投资: 向在 SuperVM 上构建的高质量 DApp 进行投资
- 基础设施: 节点补贴、RPC 服务、关键组件运维
- 流动性管理: 向 AMM 池注入或回收流动性

### 4.4 二级市场与利率曲线

- 债券可在链上自由交易(如 Bond/USDT、Bond/SVM 交易对)
- 债券价格随市场利率与信用预期变化而波动
- 长期看,SuperVM 将形成一条**链上利率曲线**,为整个生态的融资/投资提供定价基准

### 4.5 风险管理

- 保守发行: 债券利息成本需被协议现金流多倍覆盖
- 债券保险金: 发行额中提取小比例,作为潜在重组/违约缓冲
- 紧急治理: 在极端情况下,可触发延期兑付/再融资等治理流程,所有决策链上公开

## 5. CDP 系统与原生稳定币

### 5.1 抵押债仓(CDP)架构

CDP 系统允许用户锁定不同类型的抵押品,借出:

- 与 USD 软锚定的原生稳定币(名称待定,例如:某个 4 字中文名 + 对应英文 Ticker)
- 或额外的主链 Token(用于杠杆/流动性需求)

支持的抵押层级(示例):

- **Level 1: 主链 Token**  
  - 抵押率: 150%  
  - 清算线: 130%

- **Level 2: SVM 债券**  
  - 抵押率: 120%  
  - 清算线: 110%

- **Level 3: 主流加密资产(BTC/ETH)**  
  - 抵押率: 200%  
  - 清算线: 150%

- **Level 4: RWA 资产(房产/应收账款等)**  
  - 抵押率: 250%  
  - 清算线: 180%

### 5.2 稳定币设计

- 锚定目标: 1 单位 ≈ 1 USD 购买力
- 支撑方式: 以上各类抵押品形成的**超额抵押债务仓**
- 稳定费(Stability Fee): 借款人按年化利率支付给协议国库
- 清算机制: 当抵押率跌破清算线,由协议发起拍卖,罚金部分用于销毁与风险基金

### 5.3 与 MakerDAO 的对比(对外口径)

- 抵押资产更丰富: 除 BTC/ETH 外,还包括主链 Token 与 SVM 债券
- 与“主链债券 + 双轨定价”深度耦合,货币政策工具更多
- 稳定币系统与国库、债券、NAV 联动,形成闭环的经济调节体系

## 6. RWA 现实资产上链

### 6.1 资产类型

SuperVM 支持多类现实世界资产的代币化(RWA):

- 房地产(商业/住宅)
- 上市股票与部分非上市股权
- 黄金及其他贵金属
- 应收账款/发票
- 其它有稳定现金流或可评估价值的资产

### 6.2 代币形态与用途

- **RWA-Token(可分割)**: 适用于黄金、指数篮子、部分股票敞口等可拆分资产
- **RWA-NFT/1155**: 适用于单套房产、单笔应收账款等不可分或少量份额资产

在 SuperVM 体系中,RWA 具有三重角色:

1. **可交易资产**: 通过 `RWA/USDT`、`RWA/SVM` 等交易对,像股票或基金一样在链上交易
2. **抵押品**: 作为 CDP 系统中的抵押物,铸造稳定币或借出主链 Token
3. **收益载体**: 承接房租、利息、分红等现金流,按合约分配给 RWA 持有人

### 6.3 外部协议与原生整合

- 外部协议桥接: 与 Centrifuge、Maple、RealT、Backed 等 RWA 协议建立跨链桥
- 原生 RWA 上链:
  - 法律协议与资产托管安排
  - 第三方审计与估值机构
  - 上链铸造 RWA-Token/NFT,并接入 CDP 与交易系统

### 6.4 风险与法律框架

- 第三方审计: 资产真实存在性、权属与估值的定期审计
- 估值模型: 市场价、评估价、折现现金流等组合
- 司法管辖区: 如新加坡、瑞士等具备明确加密与 RWA 监管框架的地区
- 执行机制: 质押权、止赎与清算流程的法律可执行性

## 7. 收入分配与激励机制

### 7.1 80/20 收入分配

协议总收入(包括 Gas 费、手续费、稳定费、清算罚金等)按照大致 **80/20 原则** 分配:

- 约 **80%** 注入分红池,按持币比例分配给 Token 持有者
- 约 **20%** 归基金会国库,用于研发、运营与生态扩展

### 7.2 每日分红机制

- 每日固定时间(如 UTC 00:00)进行全网持仓快照
- 用户通过 on-chain `claim()` 函数主动领取分红
- 未领取部分会自动累积,未来任意时间可补领

该机制避免了“给每个地址主动发放分红”导致的 Gas 浪费,同时强化了长期持有者的收益预期。

### 7.3 外币支付下的矿工激励

- 外部链用户使用 SuperVM 服务时,支付 BTC/ETH/USDT 等外币
- 100% 外币收入进入储备池,不直接发给矿工
- 国库按 AMM 汇率,用主链 Token 向矿工/验证者支付等值报酬
- 矿工可以选择继续持有 Token(享受分红+通缩),或通过 AMM 兑换回外币

这样,外币持续沉淀为储备,而 Token 成为系统内统一的结算与激励单位。

## 8. 治理结构与货币政策

### 8.1 治理架构

- 9 人加权议会: 创始人、Top 持币者、团队代表、独立成员等构成
- 投票权按角色与持币量加权
- 支持小户委托投票(Delegation),提高参与度

### 8.2 可治理参数

- 债券发行/回购节奏与规模
- CDP 参数(抵押率、清算罚金、稳定费利率)
- NAV 赎回规则(每日限额、结算周期、阈值)
- 回购与销毁计划

### 8.3 紧急控制机制

- 极端波动下的熔断与限流(如暂时关闭 NAV 赎回)
- 特定功能的临时暂停(须有严格的时间锁与公开记录)
- 所有紧急决策必须链上记录,并受时间/权限的双重约束

## 9. 风险披露

- 市场风险: Token 价格波动、流动性枯竭
- 信用风险: 债券违约、CDP 大规模清算、RWA 对手方违约
- 技术风险: 合约漏洞、预言机故障、跨链桥攻击
- 监管风险: 各司法辖区对债券、稳定币与 RWA 的法律政策变化

SuperVM 通过: 保守抵押率、透明的链上操作、第三方审计、模块化架构等方式,尽量降低上述风险,但不会消除一切风险。

## 10. 路线图与展望

- **Phase T1**: 主链 Token、国库、外币支付替代、分红与治理核心模块
- **Phase T2**: AMM 与储备池、双轨定价、NAV 计算器、刚性兑付
- **Phase T2.5**: SVM Bonds 债券发行与链上债券市场
- **Phase T3**: 分红与治理前端 DApp
- **Phase T3.5**: CDP 系统与 RWA 整合
- **Phase T4**: 资源定价预言机与经济指标看板

SuperVM 的经济设计目标是: 用 **十年以上的时间尺度**,构建一个兼具安全性、可持续性和扩展性的去中心化金融基础设施,让链上与链下资产在同一套可编程规则下协同运作,服务全球开发者、用户与机构。
