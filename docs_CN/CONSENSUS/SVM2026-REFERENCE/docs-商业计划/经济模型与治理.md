    ## 1. 核心设计理念 <a id="核心设计理念"></a>

    SuperVM 主链 Token 经济模型设计遵循以下核心原则:

    ### 🎯 设计目标

    | 目标 | 实现机制 |
    |------|---------|
    | **长期通缩** | Gas BaseFee 自动销毁 + 手续费部分销毁 + 回购销毁 |
    | **价值锚定** | 资源定价(CPU/存储/带宽) + 多币种外汇储备 |
    | **公平分配** | 80% 收入返还持币者 + 按持仓比例每日分红 |
    | **去中心化治理** | 9人加权议会 + 小户委托投票 |
    | **多资产背书** | BTC/ETH/USDT/BNB/SOL 等外汇储备池 |
    | **市场化定价** | AMM x·y=k 恒定乘积公式 |

    ### 🌟 核心创新点

    1. **双轨收入**: 本币收入(主网服务) + 外币收入(跨链服务)
    2. **储备替代**: 外币收入归集储备,用本币支付矿工,矿工可选持有或兑换
    3. **每日分红**: 主动领取机制,避免Gas浪费,培养持有习惯
    4. **加权治理**: 创始人35% + Top5持币者50% + 团队10% + 独立5%,平衡控制权与去中心化

## 2. Token 发行与货币层级 <a id="token-发行与货币层级"></a>

### 💵 初始定价策略

**为什么不 1:1 锚定 USD/RMB?**

```
❌ 硬锚定的致命问题:
1. 储备压力: 需要 1 亿美元储备(不现实)
2. 无法上涨: 生态增长价值无法捕获
3. 脱锚风险: 参考 LUNA/UST 崩盘(2022年蒸发400亿)
4. 激励缺失: 早期参与者/团队没有收益

✅ 市场化浮动定价:
1. 低价启动: $0.05 - $0.1 (100-400x 上涨空间)
2. 储备需求小: 仅需 25-100 万美元初始流动性
3. 捕获增长: 价格随生态自然上涨
4. AMM 定价: x·y=k 透明,无需人工托市
```

**推荐方案**: 

| 方案 | 初始价格 | 初始流动性 | Year 5 目标 | 上涨空间 |
|------|---------|-----------|------------|---------|
| **低价启动** | $0.05 | 5M Token + $250K USDT | $5 | 100x |
| 稳健启动 | $0.5 | 5M Token + $2.5M USDT | $10 | 20x |

**软锚定**: 不锚定 USD,而是锚定真实资源价值

```
资源定价参考:
1 Token ≈ 1000 CPU 秒 (动态调整)
1 Token ≈ 10 GB·小时存储
1 Token ≈ 100 GB 带宽

价格形成机制:
用户需求 + 矿工供给 + AMM 池 → 市场化价格
协议仅提供资源价值参考,不强制兑付
```

### M0: 基础货币(Base Money)

```
总量上限: 100,000,000 Token (1亿枚,永不增发)
一次性铸造,分配至国库/奖励/团队/生态各账户
```

**初始分配方案**:    | 用途 | 数量 | 占比 | 解锁规则 |
    |------|------|------|---------|
    | **矿工激励池** | 40,000,000 | 40% | 10年线性释放 |
    | **生态基金** | 20,000,000 | 20% | 治理控制释放 |
    | **团队预留** | 15,000,000 | 15% | 2年锁定,之后3年线性 |
    | **早期投资者** | 10,000,000 | 10% | 1年锁定,之后2年线性 |
    | **国库储备** | 10,000,000 | 10% | 用于支付矿工外币等值Token |
    | **流动性引导** | 5,000,000 | 5% | 立即可用,注入AMM池 |

    ### M1: 流通货币(Circulating Money)

    ```
    M1 = M0已解锁部分 + 手续费重新分配 - 已销毁数量
    ```

    **M1 来源**:
    - M0 解锁(按时间表自动释放)
    - 手续费收入的矿工/持币者部分(再分配进入流通)
    - 国库支付给矿工的Token(外币收入替代)

    **M1 减少**:
    - Gas BaseFee 自动销毁
    - 手续费部分销毁
    - 国库回购销毁

    ### M2: 信用货币(Credit Money)

    ```
    M2 = M1 + 抵押稳定币 + 信用合约
    ```

    **M2 组成**:
    - **超额抵押稳定币**: 150%抵押率铸造,用于支付/交易
    - **借贷合约**: 流动性挖矿,资金利用率提升
    - **衍生品**: 期权/期货/结构化产品

    **风控机制**:
    - 实时监控抵押率,低于安全线自动清算
    - 清算罚金部分销毁,部分进风险准备金
    - M2/M1 比率预警(>3.0 触发紧缩政策)

    ## 3. 收入分配机制 <a id="收入分配机制"></a>

    ### 📊 本币收入(MainnetToken 支付)

    **来源**: 主网用户使用 Gas/交易手续费/合约执行费

    ```
    每笔 Gas 费用分配:
    ├─ 60% → 矿工节点(直接激励)
    ├─ 30% → 分红池(80%收入原则,30%+50%销毁后的替代)
    ├─ 10% → BaseFee 自动销毁(通缩机制)
    └─ 0%  → 基金会(基金会收入来自手续费)

    每笔手续费分配:
    ├─ 50% → 分红池(持币者共享)
    ├─ 20% → 基金会国库(治理控制)
    ├─ 20% → 矿工节点(额外奖励)
    └─ 10% → 自动销毁(通缩机制)
    ```

    **80/20 原则实现**:

    ```
    总收入 = Gas费 + 手续费
    分红池收入 = Gas费×30% + 手续费×50%
                ≈ 总收入 × 80% (动态调整确保)

    基金会收入 = 手续费×20%
                ≈ 总收入 × 20%
    ```

    ### 🌍 外币收入(BTC/ETH/USDT 等支付)

    **来源**: 外部链用户使用 SuperVM 节点服务

    ```
    外币支付流程:
    1. 用户支付 BTC/ETH/USDT 等
    └─ 100% 归集到外汇储备池

    2. 国库按 AMM 汇率计算等值 Token
    └─ 支付给提供服务的矿工

    3. 矿工获得 Token 后选择:
    ├─ 持有 → 享受每日分红权益
    └─ 兑换 → 通过 AMM 兑回外币
    ```

    **关键设计**:
    - ✅ 外币只进不出(除非矿工主动兑换)
    - ✅ 储备池持续增厚,Token 多资产背书
    - ✅ 矿工自由选择,市场化汇率保护双方

                        ### 🎯 双轨定价机制 <a id="双轨定价机制"></a>

                        **核心创新**: 用户可选择两种退出方式:

                        #### 价格体系

                        ```
                        Price A: 净资产价格 (NAV - Net Asset Value)
                        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                        计算公式:
                        NAV = (BTC储备×BTC价 + ETH储备×ETH价 + USDT储备 + ...) / Token流通量

                        示例:
                        外汇储备总价值: $1,600 万
                        流通量: 50,000,000 Token
                        NAV = $16,000,000 / 50,000,000 = $0.32/Token

                        特点:
                        ✓ 基于真实储备,透明可验证
                        ✓ 提供价格下限(刚性兑付保底)
                        ✓ 类似封闭式基金净值

                        Price B: 市场价格 (Market Price)
                        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                        决定因素:
                        AMM 池供需关系 (x·y=k)

                        示例:
                        Token/USDT 池: 10M Token × 5M USDT = 50M k
                        市场价 = 5M / 10M = $0.5/Token

                        特点:
                        ✓ 实时交易,即时成交
                        ✓ 可能溢价(生态繁荣时)
                        ✓ 可能折价(恐慌抛售时)
                        ```

                        #### 用户选择策略

                        ```
                        场景 1: 市场价 > NAV (溢价状态)
                        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                        示例: NAV = $0.32, 市场价 = $0.5 (56%溢价)

                        选择 AMM 交易:
                        ✓ 收益: 10,000 Token × $0.5 = $5,000
                        ✓ 时间: 即时
                        ✓ 优势: 多赚 $1,800 (56%溢价)

                        场景 2: 市场价 < NAV (折价状态)
                        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                        示例: NAV = $0.32, 市场价 = $0.2 (38%折价)

                        选择刚性兑付:
                        ✓ 收益: 10,000 Token × $0.32 = $3,200
                        ✓ 时间: T+7 天
                        ✓ 优势: 避免损失 $1,200 (保本)

                        场景 3: 套利机会
                        ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                        当市场价显著低于 NAV:

                        套利者操作:
                        1. AMM 池买入 Token (市场价 $0.2)
                        2. 向国库申请刚性兑付 (NAV $0.32)
                        3. 无风险收益: $0.12/Token = 60%

                        结果:
                        → 套利者买入 → 市场价上涨
                        → 最终市场价 ≈ NAV (套利消失)
                        → 双轨价格自然收敛!
                        ```

                        #### 刚性兑付机制

                        ```rust
                        /// 国库 NAV 赎回功能
                        pub fn redeem_at_nav(
                            user: Address,
                            token_amount: u128,
                            preferred_currency: ForeignCurrency, // USDT/BTC/ETH
                        ) -> Result<RedemptionReceipt> {
                            // 1. 计算当前 NAV
                            let nav = calculate_nav()?;
    
                            // 2. 计算应得外币价值
                            let usd_value = token_amount * nav;
    
                            // 3. 转换为指定币种
                            let amount = usd_value / get_oracle_price(preferred_currency);
    
                            // 4. 从储备池转出
                            transfer_from_reserve(preferred_currency, user, amount)?;
    
                            // 5. 销毁用户的 Token
                            burn(user, token_amount)?;
    
                            // 6. 记录赎回
                            Ok(RedemptionReceipt {
                                user,
                                token_burned: token_amount,
                                currency: preferred_currency,
                                amount_received: amount,
                                nav_price: nav,
                                timestamp: now(),
                                settlement_date: now() + 7 days, // T+7 到账
                            })
                        }

                        /// 实时 NAV 计算
                        pub fn calculate_nav() -> Result<f64> {
                            let btc_value = btc_reserve * get_oracle_price(BTC);
                            let eth_value = eth_reserve * get_oracle_price(ETH);
                            let usdt_value = usdt_reserve;
                            let bnb_value = bnb_reserve * get_oracle_price(BNB);
                            let sol_value = sol_reserve * get_oracle_price(SOL);
    
                            let total_reserve_usd = btc_value + eth_value + usdt_value + bnb_value + sol_value;
                            let circulating_supply = get_circulating_supply();
    
                            Ok(total_reserve_usd / circulating_supply)
                        }
                        ```

    ### 📈 储备池动态调整

    **AMM 恒定乘积公式**: `x · y = k`

    ```
    假设 Token/BTC 池:
    初始: 5,000,000 Token × 10 BTC = 50,000,000 k

    用户兑换 5 Token → BTC:
    新余额: (5,000,000 + 5) Token × (10 - Δ) BTC = 50,000,000
    解方程: Δ = 10 - 50,000,000/(5,000,005) ≈ 0.00001 BTC

    汇率自动调整:
    兑换前: 1 BTC = 500,000 Token
    兑换后: 1 BTC = 500,000.5 Token (Token 微贬)
    ```

    **储备健康指标**:

    ```
    储备率 (Reserve Ratio) = 当前储备 / 累计收入
    目标: RR > 80% (矿工持有倾向强)

    持有倾向 (Hold Preference) = 1 - (累计兑换 / 累计收入)
    目标: HP > 70% (矿工看好生态)
    ```

## 4.1. 主链债券市场 <a id="主链债券市场"></a>

### 🏦 SuperVM 主链债券 (SVM Bonds)

**核心功能**:
1. 国库融资工具(不稀释 Token)
2. 货币政策工具(调节流通量)
3. 信用扩张基础(抵押品)

#### 债券设计

```
债券参数:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
面值: 100 USDT
发行价: 95 USDT (5% 折价)
期限: 1年/3年/5年
票面利率: 8%/12%/15% (年化)
利息支付: 每季度支付 Token
本金偿还: 到期一次性 USDT

示例 - 1年期债券:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
购买成本: 95 USDT
年利息: 100 × 8% = 8 USDT (按季度付 Token)
到期收回: 100 USDT
总收益: (100 + 8) - 95 = 13 USDT
实际年化: 13 / 95 = 13.68%
```

#### 债券用途

```
1. 国库融资
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
发行规模: 1000 万 USD (Year 1)

用途分配:
├─ 生态投资: 500 万 (投资优质 DApp)
├─ 基础设施: 300 万 (节点补贴、RPC 服务)
└─ 流动性管理: 200 万 (AMM 池注入)

优势:
✓ 不稀释 Token 持有者权益
✓ 固定成本 (8-15% 利息 vs 不确定的增发)
✓ 吸引保守投资者(稳定收益产品)

2. 货币政策工具
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
通胀时(Token价格下跌):
→ 发行债券 → 回收 Token → 减少流通 → 价格支撑

通缩时(Token价格暴涨):
→ 回购债券 → 释放 Token → 增加流通 → 价格稳定

类似: 美联储公开市场操作 (OMO)

3. 信用扩张基础
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
债券作为抵押品:
用户抵押 SVM Bond → 借出 Token
抵押率: 120% (债券比 Token 更稳定)

示例:
持有 100 万 USDT 面值债券
→ 可借出 100万/120% = 83.3万 USDT 等值 Token
→ 用于流动性挖矿/质押
→ 债券利息持续获得(双重收益)
```

```
二级市场债券价格:

| **价格发现** | 集中撮合,有涨跌停等限制 | AMM 池 + 订单簿(可选),无涨跌停,由供需和预言机共同决定 | SuperVM 不直接干预价格,只提供基础设施 |

| **清算方式** | 违约时由券商/法院处置股票 | 抵押率低于清算线时,由协议自动发起拍卖/回购 | 清算规则提前写死在合约中,可审计、可预测 |
| **交易层风险** | 市场波动、流动性不足 | 市场波动、AMM 滑点 | 两者在“价格波动”层面是同类风险 |

从这个对比可以看出:

- **在交易层**,RWA 完全可以“像股票一样”挂在市场上做二级交易;
- **在抵押层**,RWA 又可以“像房产/股票质押”一样释放流动性;

    ## 5. 分红领取规则 <a id="分红领取规则"></a>

    ### 💎 每日主动领取机制

    **设计理由**:
    - ❌ 自动分红: 每日数万/百万地址,Gas 成本高昂
    - ✅ 主动领取: 用户自主决定,节省 Gas,培养持有周期性

    **领取规则**:

    ```rust
    每日快照时间: UTC 00:00
    可领取周期: 24小时
    累计机制: 未领取自动累加到下一期

    计算公式:
    用户每日可领分红 = (用户持仓 / 总流通量) × 当日分红池收入

    示例:
    - 用户持有: 10,000 Token
    - 总流通量: 50,000,000 Token
    - 当日分红池收入: 100,000 Token
    - 用户可领取: (10,000 / 50,000,000) × 100,000 = 20 Token
    ```

    ### 📅 领取流程

    ```
    ┌─────────────────────────────────────────────────┐
    │  Day 1 (11月17日)                                │
    │  - 00:00 快照持仓                                │
    │  - 分红池收入: 100,000 Token                     │
    │  - 用户持仓占比: 0.02%                           │
    │  - 可领取: 20 Token                              │
    ├─────────────────────────────────────────────────┤
    │  Day 2 (11月18日)                                │
    │  - 用户未领取 Day 1 分红                         │
    │  - 分红池收入: 120,000 Token                     │
    │  - 累计可领取: 20 + 24 = 44 Token                │
    ├─────────────────────────────────────────────────┤
    │  Day 3 (11月19日)                                │
    │  - 用户领取全部累计: 44 Token                    │
    │  - 新一期开始计算                                │
    └─────────────────────────────────────────────────┘
    ```

    ### 🔒 防滥用机制

    1. **快照时持仓**: 转账在快照后不影响当期分红
    2. **一次性领取**: 每期最多领取一次,防止合约重入攻击
    3. **Gas 优化**: Merkle Tree 证明,节省链上存储
    4. **最小持仓**: < 100 Token 不参与分红(防粉尘攻击)

    - 独立委员(5%): ❌ 反对

    总赞成票: 35% + 30% + 10% = 75%
    状态: ✅ 通过(>50% 阈值)
    ```

    ## 7. AMM 定价机制 <a id="amm-定价机制"></a>

    ### 🔄 恒定乘积公式

    **Uniswap V2 模型**: `x · y = k`

    ```
    Token 池: x Token
    USD-Stable 池: y USDT
    恒定乘积: k = x × y

    价格计算:
    P(Token/USDT) = y / x

    示例:
    初始池子: 10,000,000 Token × 1,000,000 USDT
    初始价格: 1,000,000 / 10,000,000 = $0.1/Token

    用户买入 10,000 Token:
    新余额: (10,000,000 - 10,000) × (1,000,000 + ΔUSDT) = 100,000,000,000,000
    解方程: ΔUSDT = 100,000,000,000,000 / 9,990,000 - 1,000,000 ≈ 1,001 USDT
    实际支付: 1,001 USDT (含滑点)
    新价格: 1,001,001 / 9,990,000 ≈ $0.10021/Token
    ```

    ### 💹 多池子汇率联动

    ```
    Token/USDT 池: 主定价池
    Token/BTC 池: 通过 USDT 间接定价
    Token/ETH 池: 通过 USDT 间接定价

    套利机制:
    如果 Token/BTC 池价格偏离 Token/USDT × USDT/BTC:
    → 套利者买低卖高
    → 价格自动收敛
    ```

    ### 📊 流动性挖矿激励

    **目标**: 引导早期流动性,稳定价格

    ```
    LP 奖励分配:
    - Token/USDT 池: 40% 奖励(主池优先)
    - Token/BTC 池: 20% 奖励
    - Token/ETH 池: 20% 奖励
    - 其他池: 20% 奖励

    年化收益(APY):
    APY = (年奖励 Token 价值 / 池子总流动性) × 100%

    示例:
    Token/USDT 池流动性: $10,000,000
    年奖励: 1,000,000 Token (价值 $100,000)
    APY = $100,000 / $10,000,000 × 100% = 1%
    + 交易手续费 0.3% 分成 ≈ 年化 5-10%
    总 APY ≈ 6-11%
    ```

    ## 8. 通缩与价值增长 <a id="通缩与价值增长"></a>

    ### 🔥 多维度销毁机制

    | 销毁类型 | 触发条件 | 销毁量 | 年化影响 |
    |---------|---------|-------|---------|
    | **Gas BaseFee** | 每笔交易自动 | 10-15% Gas费 | -2% 供应量 |
    | **手续费销毁** | 每笔手续费 | 10% | -0.5% 供应量 |
    | **清算罚金** | M2抵押不足 | 罚金的50% | -0.3% 供应量 |
    | **国库回购** | 治理决策 | 不定期 | -1% 供应量 |

    **预期通缩率**: 年化 -3% ~ -5%

    ### 📈 价值增长路径

    ```
    年份 1-2: 基础设施建设期
    - 目标: 稳定在 $0.1-0.5
    - 策略: 大量 LP 激励,吸引用户
    - 流通量: 20-30M Token

    年份 3-5: 生态爆发期
    - 目标: 突破 $1,达到 $2-5
    - 策略: DApp 繁荣,跨链服务增长
    - 流通量: 40-50M Token(销毁抵消解锁)

    年份 6-10: 价值稳定期
    - 目标: 稳定在 $5-20
    - 策略: 外汇储备深厚,多资产背书
    - 流通量: 50-60M Token(强通缩)
    ```

    ### 🎯 超越 USD 的路径

    1. **资源锚定**: Token 直接购买 CPU/存储/带宽,需求刚性
    2. **分红吸引**: 年化 10-20% 分红,吸引长期持有
    3. **外汇背书**: BTC/ETH 储备支撑,类似数字央行
    4. **通缩螺旋**: 供应减少 + 需求增加 → 价格上涨
    5. **网络效应**: 用户越多 → 收入越高 → 分红越多 → 吸引更多用户

    **数学模型**:

    ```
    Token 价值 = f(资源需求, 分红收益, 外汇储备, 稀缺性)

    资源需求 ∝ 活跃 DApp 数量 × 平均资源消耗
    分红收益 = 年收入 × 80% / 流通量
    外汇储备 = Σ(各币种储备 × 各币种价格)
    稀缺性 = 1 / 流通量

    当:
    - DApp 数量 10x (10倍增长)
    - 分红率 > 银行利率(15% vs 3%)
    - 外汇储备 > 市值 50%
    - 通缩率 -5%/年

    则:
    Token 价格 > USD 增长率 (长期)
    ```
