# NOVOVM 命名冻结规范（2026-03-03）

## 1. 品牌层级

- 对外品牌：`NOVOVM`
- 技术简称：`NVM`
- 执行内核名称：`AOEM Engine`
- `SuperVM`：仅保留为内部历史代号，不再对外主打

## 2. 对外文档规则

- 官网、白皮书、发布文档统一使用 `NOVOVM`。
- 涉及执行层时统一写 `Powered by AOEM Engine`。
- 新增对外文档禁止把 `SuperVM` 当作产品名使用。

## 3. 白皮书命名

- 标题：`NOVOVM Whitepaper`
- 副标题：`Powered by AOEM Execution Engine`

## 4. 代码命名基线

- 新增 crate 统一前缀 `novovm-`：
  - `novovm-node`
  - `novovm-cli`
  - `novovm-exec`
  - `novovm-prover`
- 历史 `supervm-*` 仅做迁移兼容，逐步退役。

## 5. 命令命名

- 用户命令统一为 `novovm`：
  - `novovm start`
  - `novovm status`
  - `novovm metrics`
  - `novovm version`

## 6. 迁移策略

- 允许内部路径暂时存在 `SUPERVM` 目录名，不影响对外品牌统一。
- 迁移期内保持“外部命名已统一，内部代码逐步重命名”的双轨策略。
