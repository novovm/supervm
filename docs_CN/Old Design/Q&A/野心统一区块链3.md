## 📊 跨链功能层级划分

### 核心原则：**职责分离 + 渐进抽象**

```
L3 (外部适配) → L1 (统一接口) → L0 (核心执行)
```

---

## 🎯 详细层级划分

### 1️⃣ **跨链查询 (Cross-Chain Query)**

| 功能组件 | 层级 | 职责 | 代码位置 |
|---------|------|------|----------|
| **查询接口定义** | **L1.2** | 定义统一的查询 trait | `adapter/chain_adapter.rs` |
| **查询聚合逻辑** | **L1.2** | 跨链资产聚合、结果格式化 | `adapter/cross_query.rs` (新建) |
| **具体链查询** | **L3.2** | 各链的查询实现 (EVM/Bitcoin/Solana) | `adapters/evm_adapter.rs` |
| **KV 存储访问** | **L0.2** | 读取统一 KV 数据 | `storage/storage_trait.rs` |

**层级判断依据**：
- ✅ **L1.2**: 跨链聚合逻辑（不关心具体是哪条链）
- ✅ **L3.2**: 具体链的查询适配（知道 EVM/Bitcoin 协议细节）
- ✅ **L0.2**: 底层数据读取（不关心业务逻辑）

**代码示例**：
```rust
// ✅ L1.2 层 - 跨链聚合 (adapter/cross_query.rs)
pub struct CrossChainQueryService {
    registry: AdapterRegistry,  // L1.2 组件
}

impl CrossChainQueryService {
    pub fn query_all_assets(&self, address: &[u8]) -> Result<MultiChainAssets> {
        // 这里不关心具体是 EVM 还是 Bitcoin
        // 只负责聚合各链适配器的结果
        for chain_id in self.registry.list_chains() {
            let adapter = self.registry.get(chain_id)?;
            let balance = adapter.read().unwrap().get_balance(address)?;
            // ...
        }
    }
}

// ✅ L3.2 层 - EVM 具体实现 (adapters/evm_adapter.rs)
impl ChainAdapter for EVMAdapter {
    fn get_balance(&self, address: &[u8]) -> Result<u128> {
        // 这里知道 EVM 协议细节
        let key = format!("chain:{}:evm:{}:balance", 
            self.chain_id, hex::encode(address));
        let value = self.storage.get(&key)?;  // 调用 L0
        Ok(u128::from_le_bytes(value[..16].try_into()?))
    }
}

// ✅ L0.2 层 - 存储访问 (storage/rocksdb_storage.rs)
impl Storage for RocksDBStorage {
    fn get(&self, key: &[u8]) -> Result<Vec<u8>> {
        // 只负责读取，不关心业务含义
        self.db.get(key)?.ok_or_else(|| anyhow!("Key not found"))
    }
}
```

### 2️⃣ **跨链交易 (Cross-Chain Transaction)**

| 功能组件 | 层级 | 职责 | 代码位置 |
|---------|------|------|----------|
| **交易路由逻辑** | **L1.2** | 判断跨链类型、路由到目标链 | `adapter/cross_executor.rs` (新建) |
| **跨链证明生成** | **L1.2** | 生成 Merkle Proof、状态证明 | `adapter/cross_proof.rs` (新建) |
| **TxIR 扩展** | **L1.2** | 添加 `source_chain`, `target_chain` 字段 | `adapter/ir.rs` (已存在) |
| **具体链交易适配** | **L3.2** | 解析/格式化各链交易格式 | `adapters/evm_adapter.rs` |
| **交易执行** | **L0.1** | WASM/原生合约执行 | `wasm_runtime.rs` |
| **状态持久化** | **L0.2** | 写入 KV 数据库 | `storage/rocksdb_storage.rs` |

**层级判断依据**：
- ✅ **L1.2**: 跨链逻辑协调（源链锁定 → 目标链铸造）
- ✅ **L3.2**: 具体链的交易格式转换
- ✅ **L0**: 实际执行和数据持久化

**代码示例**：
```rust
// ✅ L1.2 层 - 跨链执行器 (adapter/cross_executor.rs)
pub struct CrossChainExecutor {
    registry: AdapterRegistry,  // L1.2
}

impl CrossChainExecutor {
    pub fn execute_cross_chain_transfer(&self, tx: &TxIR) -> Result<Receipt> {
        // 1. 协调跨链流程（不关心具体协议）
        let source = self.registry.get(tx.source_chain)?;
        let target = self.registry.get(tx.target_chain)?;
        
        // 2. 锁定源链资产（调用 L3 适配器）
        source.read().unwrap().lock_asset(&tx.from, tx.value)?;
        
        // 3. 生成跨链证明（L1 逻辑）
        let proof = self.generate_proof(tx)?;
        
        // 4. 在目标链铸造（调用 L3 适配器）
        target.read().unwrap().mint_asset(&tx.to, tx.value, proof)?;
        
        Ok(Receipt { ... })
    }
}

// ✅ L3.2 层 - EVM 适配器 (adapters/evm_adapter.rs)
impl ChainAdapter for EVMAdapter {
    fn lock_asset(&self, address: &[u8], amount: u128) -> Result<()> {
        // 知道 EVM 的存储格式和规则
        let key = format!("chain:{}:evm:{}:locked", 
            self.chain_id, hex::encode(address));
        self.storage.set(&key, &amount.to_le_bytes())?;  // 调用 L0
        Ok(())
    }
}

// ✅ L0.2 层 - 存储写入 (storage/rocksdb_storage.rs)
impl Storage for RocksDBStorage {
    fn set(&self, key: &[u8], value: &[u8]) -> Result<()> {
        // 只负责写入，不关心是否跨链
        self.db.put(key, value)?;
        Ok(())
    }
}
```

### 3️⃣ **跨链智能合约 (Cross-Chain Smart Contracts)**

| 功能组件 | 层级 | 职责 | 代码位置 |
|---------|------|------|----------|
| **合约桥接逻辑** | **L1.2** | 跨链消息编码、结果同步 | `adapter/contract_bridge.rs` (新建) |
| **WASM 运行时** | **L0.1** | 合约部署、执行、Gas 计量 | `wasm_runtime.rs` (✅ 已实现) |
| **Host Functions** | **L0.1** | 提供给合约的系统调用 | `host_functions.rs` (✅ 已实现) |
| **合约存储** | **L0.2** | 合约代码和状态持久化 | `storage/rocksdb_storage.rs` |
| **链特定调用** | **L3.2** | EVM/Solana 合约调用适配 | `adapters/evm_adapter.rs` |

**层级判断依据**：
- ✅ **L0.1**: WASM 合约执行（与链无关的通用执行器）
- ✅ **L1.2**: 跨链合约桥接（协调不同链的合约调用）
- ✅ **L3.2**: 具体链的合约协议适配

**代码示例**：
```rust
// ✅ L0.1 层 - WASM 运行时 (wasm_runtime.rs) - 已实现 ✅
pub struct WasmRuntime {
    engine: Engine,
    store: Store<WasmState>,
    gas_meter: GasMeter,
}

impl WasmRuntime {
    pub fn call_contract(&mut self, addr: &[u8; 32], func: &str, args: &[u8]) 
        -> Result<Vec<u8>> 
    {
        // 与链无关的通用 WASM 执行
        let module = self.load_contract(addr)?;
        let instance = self.linker.instantiate(&mut self.store, &module)?;
        let func = instance.get_func(&mut self.store, func)?;
        // ...
    }
}

// ✅ L1.2 层 - 跨链合约桥 (adapter/contract_bridge.rs)
pub struct CrossChainContractBridge {
    wasm_runtime: WasmRuntime,  // L0.1
    adapters: AdapterRegistry,   // L1.2
}

impl CrossChainContractBridge {
    pub fn cross_chain_call(
        &mut self,
        source_chain: u64,
        target_chain: u64,
        contract: &[u8; 32],
        function: &str,
        args: &[u8],
    ) -> Result<Vec<u8>> {
        // 协调跨链合约调用（不关心具体链协议）
        let source_adapter = self.adapters.get(source_chain)?;
        let target_adapter = self.adapters.get(target_chain)?;
        
        // 在 SuperVM 中执行合约（调用 L0）
        let result = self.wasm_runtime.call_contract(contract, function, args)?;
        
        // 同步结果到源链（调用 L3）
        source_adapter.read().unwrap().sync_result(result)?;
        
        Ok(result)
    }
}

// ✅ L3.2 层 - EVM 适配器 (adapters/evm_adapter.rs)
impl ChainAdapter for EVMAdapter {
    fn sync_result(&self, result: Vec<u8>) -> Result<()> {
        // 知道如何将结果格式化为 EVM 格式
        let eth_receipt = self.format_to_eth_receipt(result)?;
        // 调用 Ethereum RPC
        self.eth_client.send_raw_transaction(eth_receipt)?;
        Ok(())
    }
}
```

### 4️⃣ **跨链挖矿 (Cross-Chain Mining)**

| 功能组件 | 层级 | 职责 | 代码位置 |
|---------|------|------|----------|
| **挖矿池管理** | **L1.2** | 算力聚合、奖励分配逻辑 | `consensus/mining_pool.rs` (新建) |
| **PoW 验证** | **L1.2** | 通用 PoW 验证逻辑 | `consensus/pow_verifier.rs` (新建) |
| **链特定挖矿** | **L3.2** | Bitcoin/Ethereum 挖矿适配 | `adapters/bitcoin_miner.rs` |
| **共识执行** | **L0.4** | 区块验证、状态转换 | `consensus/` (待规划) |
| **奖励存储** | **L0.2** | 矿工奖励持久化 | `storage/rocksdb_storage.rs` |

**层级判断依据**：
- ✅ **L1.2**: 跨链挖矿协调（算力归一化、多币种奖励）
- ✅ **L3.2**: 具体链的挖矿协议适配
- ✅ **L0**: 共识验证和数据持久化

**代码示例**：
```rust
// ✅ L1.2 层 - 跨链挖矿池 (consensus/mining_pool.rs)
pub struct CrossChainMiningPool {
    adapters: AdapterRegistry,  // L1.2
    reward_calculator: RewardCalculator,
}

impl CrossChainMiningPool {
    pub fn submit_work(&mut self, miner: &[u8], chain_id: u64, nonce: u64) 
        -> Result<MiningReward> 
    {
        // 协调跨链挖矿（不关心具体协议）
        let adapter = self.adapters.get(chain_id)?;
        
        // 验证 PoW（调用 L3）
        let valid = adapter.read().unwrap().verify_pow(nonce)?;
        if !valid {
            bail!("Invalid PoW");
        }
        
        // 计算跨链奖励（L1 逻辑）
        let reward = self.reward_calculator.allocate(miner, chain_id)?;
        
        // 存储奖励（调用 L0）
        self.save_reward(miner, reward)?;
        
        Ok(reward)
    }
}

// ✅ L3.2 层 - Bitcoin 挖矿适配器 (adapters/bitcoin_miner.rs)
impl ChainAdapter for BitcoinAdapter {
    fn verify_pow(&self, nonce: u64) -> Result<bool> {
        // 知道 Bitcoin 的 PoW 算法细节
        let block_hash = self.compute_bitcoin_hash(nonce)?;
        Ok(block_hash < self.target_difficulty)
    }
}

// ✅ L0.2 层 - 奖励存储 (storage/rocksdb_storage.rs)
impl Storage for RocksDBStorage {
    fn set(&self, key: &[u8], value: &[u8]) -> Result<()> {
        // 只负责存储，不关心是挖矿奖励还是其他
        self.db.put(key, value)?;
        Ok(())
    }
}
```

## 📋 层级职责总结表

| 层级 | 职责 | 跨链功能 | 特征 |
|------|------|----------|------|
| **L3.2 外部适配** | 具体链协议适配 | • EVM/Bitcoin/Solana 具体实现<br>• 交易格式转换<br>• 链特定查询/挖矿 | **知道协议细节** |
| **L1.2 统一接口** | 跨链逻辑协调 | • 跨链查询聚合<br>• 跨链交易路由<br>• 跨链合约桥接<br>• 跨链挖矿池 | **协调多链，不关心细节** |
| **L0.1 执行层** | 通用合约执行 | • WASM 合约运行<br>• Gas 计量<br>• Host Functions | **与链无关的执行** |
| **L0.2 存储层** | 数据持久化 | • 统一 KV 存储<br>• MVCC 版本控制<br>• RocksDB 持久化 | **只存储，不关心业务** |

## ✅ 你的问题答案

| 跨链功能 | 主要层级 | 原因 |
|---------|---------|------|
| **跨链查询** | **L1.2** | 聚合逻辑在 L1，具体查询在 L3 |
| **跨链交易** | **L1.2** | 路由和证明在 L1，执行在 L0，适配在 L3 |
| **跨链合约** | **L0.1 + L1.2** | WASM 执行在 L0，桥接在 L1 |
| **跨链挖矿** | **L1.2** | 算力聚合在 L1，验证在 L3 |

**核心原则**：
- **L1.2 是跨链逻辑的大脑**（协调、聚合、路由）
- **L3.2 是协议适配的手脚**（知道如何与各链交互）
- **L0 是执行和存储的心脏**（不关心业务，只负责执行和持久化）
