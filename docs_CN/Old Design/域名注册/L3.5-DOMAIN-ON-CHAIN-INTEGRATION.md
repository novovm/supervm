## 🎯 核心特性

### 1️⃣ 链上交易模型 (`DomainTx`)

```rust
pub struct DomainTx {
    pub from: String,           // 发起者（公钥哈希）
    pub nonce: u64,             // 防重放 nonce
    pub kind: DomainTxKind,     // 操作类型
    pub pubkey: Vec<u8>,        // Ed25519 公钥 (32 bytes)
    pub signature: Vec<u8>,     // Ed25519 签名 (64 bytes)
}

pub enum DomainTxKind {
    Register { name: String, years: u32, metadata: Option<DomainMetadata> },
    Transfer { name: String, new_owner: String },
    BindNode { name: String, node_id: String },
    UnbindNode { name: String, node_id: String },
    Renew { name: String, years: u32 },
}
```

**签名验证流程**:
1. 序列化消息: `from || nonce || json(kind)`
2. Ed25519 签名验证
3. Nonce 防重放检查（HashSet 追踪）
4. 转换为内部 `DomainOperation` 执行

---

### 2️⃣ 事件系统

```rust
pub enum DomainEvent {
    Registered { name: String, owner: String, expiry: u64, fee: u64 },
    Transferred { name: String, old_owner: String, new_owner: String },
    Renewed { name: String, new_expiry: u64 },
}
```

**事件发射时机**:
- 注册成功 → `Registered` 事件
- 转让成功 → `Transferred` 事件（包含旧所有者）
- 续费成功 → `Renewed` 事件（包含新过期时间）

**读取事件**:
```rust
let executor = DomainChainExecutor::new(registry);
executor.execute(tx); // 执行交易
let events = executor.drain_events(); // 获取并清空事件队列
```

### 3️⃣ SuperVM 集成

#### 路由方法
```rust
impl SuperVM {
    pub fn execute_domain_tx(&self, dt: &DomainTx) -> ExecutionReceipt {
        let receipt = self.domain_chain.execute(dt.clone());
        ExecutionReceipt {
            path: ExecutionPath::ConsensusPath, // 域名交易走共识路径
            accepted: receipt.success,
            reason: receipt.error,
            success: receipt.success,
            // ...
        }
    }
}
```

**集成点**:
- SuperVM 新增字段 `domain_chain: DomainChainExecutor`
- 初始化时自动创建内部 `DomainRegistry`
- 所有域名交易默认路由到 `ConsensusPath`（保证全局一致性）

### 4️⃣ CLI RPC 提交模式

#### 本地模式（原有功能）
```bash
# 使用内存数据库
domain-cli register myapp --owner alice --years 2

# 使用持久化数据库
domain-cli --db ./domains.db list --owner alice
```

#### RPC 模式（新增）
```bash
# 1. 生成密钥对
domain-cli keygen --output my.key
# 输出: ✅ Generated keypair saved to my.key
#       Public key: a1b2c3d4...

# 2. 提交链上注册交易
domain-cli --rpc http://localhost:8080 --keyfile my.key \
  register myapp --owner alice --years 2

# 3. 查询（通过 RPC 节点）
domain-cli --rpc http://localhost:8080 get myapp.w3
```

**RPC 端点规范**:
- **POST** `/domain/submit`
  - Body: `DomainTx` JSON
  - Response: `{ "success": true, "tx_hash": "0x..." }`

**签名流程**:
1. CLI 读取私钥文件（32 字节 Ed25519 seed）
2. 构造 `DomainTx` 并序列化消息
3. 使用私钥签名 → 64 字节签名
4. HTTP POST 到 RPC 节点
5. 节点调用 `SuperVM::execute_domain_tx`

```rust
#[test]
fn test_register_success() {
    let exec = DomainChainExecutor::new(DomainRegistry::new());
    let key = SigningKey::generate(&mut rand::rngs::OsRng);
    let tx = DomainTx::register("game", "owner1", 1, &key, 1);
    let receipt = exec.execute(tx);
    assert!(receipt.success);
    assert!(receipt.expiry.is_some());
    assert_eq!(exec.drain_events().len(), 1); // 检查事件
}

#[test]
fn test_register_duplicate() {
    // 同一域名二次注册失败
    let exec = DomainChainExecutor::new(DomainRegistry::new());
    let k1 = SigningKey::generate(&mut rand::rngs::OsRng);
    let k2 = SigningKey::generate(&mut rand::rngs::OsRng);
    assert!(exec.execute(DomainTx::register("dup", "owner1", 1, &k1, 1)).success);
    let r2 = exec.execute(DomainTx::register("dup", "owner2", 1, &k2, 2));
    assert!(!r2.success);
}

#[test]
fn test_transfer_chain_progression() {
    let exec = DomainChainExecutor::new(DomainRegistry::new());
    let ka = SigningKey::generate(&mut rand::rngs::OsRng);
    let kb = SigningKey::generate(&mut rand::rngs::OsRng);
    
    // 注册域名
    assert!(exec.execute(DomainTx::register("site", "alice", 1, &ka, 1)).success);
    
    // 正常转让
    let mut tx_transfer = DomainTx { /* ... */ };
    tx_transfer.sign(&ka);
    assert!(exec.execute(tx_transfer).success);
    
    // 重放攻击（相同 nonce）应失败
    let mut replay = DomainTx { from: "alice", nonce: 2, /* ... */ };
    replay.sign(&ka);
    assert!(!exec.execute(replay).success); // ❌ nonce 已使用
    
    // 伪造签名（错误私钥）应失败
    let mut forged = DomainTx { from: "alice", nonce: 3, /* ... */ };
    forged.sign(&kb); // 使用不同私钥
    assert!(!exec.execute(forged).success); // ❌ 签名验证失败
}
```

## 📊 性能指标

| 指标 | 数值 | 说明 |
|------|------|------|
| **签名验证延迟** | ~50-100μs | Ed25519 单次验证（纯 CPU） |
| **Nonce 检查** | O(1) | HashSet 查询 |
| **事件发射** | O(1) | Vec 追加，无序列化开销 |
| **MVCC 写入** | ~200-500μs | 含索引更新（本地 RocksDB） |
| **端到端延迟** | ~1-2ms | 签名验证 + MVCC + 事件（本地） |

**优化建议**:
- 批量验证：Ed25519 批量签名验证可提升 2-3 倍吞吐
- Nonce 裁剪：定期清理旧 nonce（如保留最近 10 万条）
- 并发执行：域名操作天然隔离，可支持 FastPath 优化

## 🔐 安全特性

### 1️⃣ 签名验证
- **算法**: Ed25519（RFC 8032）
- **抗攻击**: 抗伪造、抗重放、抗篡改
- **性能**: 50μs/签名（CPU），适合高频验证

### 2️⃣ Nonce 防重放
- **机制**: 追踪 `(from, nonce)` 对到 `HashSet`
- **拒绝条件**: 相同 `(from, nonce)` 二次提交
- **存储**: 内存哈希表（未来可持久化到 DB）

### 3️⃣ 域名唯一性
- **MVCC 检查**: 注册时快照读检查是否已存在
- **后缀自动补全**: 非注册操作自动添加 `.w3` 后缀统一命名空间

### 4️⃣ 权限校验（底层）
- **所有者验证**: `DomainRegistry::verify_signature` 检查操作者是否为当前所有者
- **当前实现**: 占位函数（返回 `Ok()`），未来集成真实公钥验证

```rust
use vm_runtime::domain_chain::{DomainChainExecutor, DomainTx};
use l0_atomic::domain_registry::DomainRegistry;

let executor = DomainChainExecutor::new(DomainRegistry::new());

// 执行多笔交易
executor.execute(tx1);
executor.execute(tx2);

// 批量获取事件
let events = executor.drain_events();
for event in events {
    match event {
        DomainEvent::Registered { name, owner, .. } => {
            println!("New domain: {} by {}", name, owner);
        }
        DomainEvent::Transferred { name, old_owner, new_owner } => {
            println!("{}: {} -> {}", name, old_owner, new_owner);
        }
        _ => {}
    }
}
```
