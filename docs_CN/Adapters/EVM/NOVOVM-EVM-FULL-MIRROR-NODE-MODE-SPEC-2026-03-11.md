# NOVOVM EVM 全功能镜像节点模式规范（SUPERVM）- 2026-03-11

## 1. 目标定义

EVM 插件在 SUPERVM 中的定位不是“轻量兼容器”，而是：

1. `EVM 全功能镜像节点`：对外提供完整节点服务能力（查询、交易、合约执行、事件面）。
2. `SUPERVM 增强节点`：复用 AOEM 与持久化等底座能力，获得高性能并发、GPU、密码学与统一治理能力。
3. `可收益节点`：具备验证者/服务节点费收取能力，并按策略归集到 SUPERVM 储备与节点账户。

## 2. 生产架构约束

1. 外部协议兼容：
   - 对外允许 `HTTP/JSON-RPC`（钱包、SDK、外部系统）。
2. 内部执行链路：
   - 统一 `二进制流水线`（D1/D2/D3/AOEM），禁止内部回退到 RPC/HTTP 传输。
3. 主入口稳定性：
   - 插件接入不得要求改动 `novovm-node` 主入口语义。
4. 持久化与执行：
   - 与 AOEM/StateDB（含 rocksdb 变体）共享底座能力，避免重复实现。

## 3. 节点收益与结算模型（对应需求 1）

### 3.1 收益来源

- EVM 节点服务费（gas/服务费）。
- 插件策略允许的验证/服务收入。

### 3.2 归集规则

1. 收入先归集到 SUPERVM 原生外汇储备账户（reserve）。
2. 按配置策略换汇到 SUPERVM 原生 token。
3. 结算到节点配置账户（视作可持续节点收益能力）。

### 3.3 接口约束

- 收益结算必须可审计、可回放、可对账。
- 结算策略必须由配置/治理控制，不允许硬编码分配路径。

## 4. 内存级策略数据路径（对应需求 2）

### 4.1 核心要求

- 策略引擎优先消费 `内存级 ingress/执行前` 数据，不依赖外部二次 RPC 拉取。
- 目标是降低外部查询延迟与额外成本，优先在节点本地完成策略判断。

### 4.2 数据面

1. 交易 ingress 内存帧（raw tx + 解析结果 + 时间戳）。
2. 执行前上下文（必要状态视图/策略信号）。
3. 批次执行反馈（成功/拒绝原因）。

### 4.3 性能口径

- 热路径禁止“每笔一日志”。
- 默认极致静默，仅保留错误与批次汇总信号。

## 5. 单机原子跨链基础（对应需求 3）

### 5.1 核心要求

- 在同一节点设备中，基于共享 AOEM + 持久化，实现跨链原子意图的本地编排。
- 本地原子检查通过后才进入广播流程；失败则整体回滚/拒绝。

### 5.2 执行原则

1. 原子意图使用统一事务上下文（intent）。
2. 跨链 legs 在本地做完整前置约束检查。
3. 不满足 all-or-nothing 条件时禁止部分广播。

## 6. 接口基线（adapter-api）

已在 `novovm-adapter-api` 预留镜像节点扩展接口（不破坏现有 `ChainAdapter` 主契约）：

- `EvmMirrorNodeAdapterExt::drain_mempool_ingress_frames`（内存 ingress 数据面）
- `EvmMirrorNodeAdapterExt::submit_atomic_cross_chain_intent`（原子跨链意图）
- `EvmMirrorNodeAdapterExt::set_fee_settlement_policy / settle_fee_income`（收益结算）

对应代码：

- `crates/novovm-adapter-api/src/evm_mirror.rs`

## 7. 阶段推进（与迁移台账对齐）

1. Stage A（当前）：
   - 完成接口契约、配置约束、台账口径与风险项。
2. Stage B：
   - 在 EVM 插件实现内存 ingress 队列与收益结算最小闭环。
3. Stage C：
   - 接入原子跨链 intent 执行器与广播门控。
4. Stage D：
   - 进入 M2/M3 全功能镜像收口与压力验证。

## 8. 非目标

- 不在内部主链路引入 RPC/HTTP 中转。
- 不为短期演示牺牲终局架构一致性。
